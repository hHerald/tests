


# Анализ эффективности YouTube Shorts

**Полноценный проект по разведочному анализу данных (EDA), статистическому тестированию гипотез и построению моделей машинного обучения для предсказания вовлечённости коротких видео на YouTube.**

## Описание проекта

В работе проводится комплексный анализ датасета из 300 коротких видео (YouTube Shorts) из различных категорий: технологии, путешествия, еда, юмор, лайфстайл и образование.

**Цель проекта** — выявить ключевые факторы, влияющие на вовлечённость аудитории, и построить предсказательную модель для коэффициента вовлечённости (`engagement_rate`).

### Основные задачи
- Проведение разведочного анализа данных (EDA)
- Визуализация распределений признаков и сравнение категорий
- Проверка статистических гипотез с помощью тестов
- Feature Engineering (создание целевой переменной и вспомогательных признаков)
- Построение, обучение и сравнение моделей машинного обучения
- Анализ важности признаков (feature importance)

## Используемые библиотеки

Для выполнения анализа и моделирования использовались следующие библиотеки Python:

- **NumPy** (`numpy`) — математические вычисления и работа с многомерными массивами
- **Pandas** (`pandas`) — загрузка, обработка и анализ табличных данных (DataFrame)
- **SciPy** (`scipy.stats`) — статистические тесты (t-тест Уэлча, Mann-Whitney U) и описательные статистики
- **Scikit-learn** (`sklearn`) — разбиение данных, модели регрессии (LinearRegression, RandomForestRegressor, GradientBoostingRegressor), метрики качества (RMSE, R²)
- **Seaborn** (`seaborn`) — статистические визуализации и красивые графики
- **Matplotlib** (`matplotlib.pyplot`) — базовые графики и настройка визуализаций

## Датасет

Датасет содержит подробные метрики эффективности 300 YouTube Shorts из разных категорий.

**Ссылка на датасет:** [YouTube Shorts Performance Dataset (Kaggle)](https://www.kaggle.com/datasets/prince7489/youtube-shorts-performance-dataset)

### Описание признаков

| Признак            | Описание                                      | Тип       |
|--------------------|-----------------------------------------------|-----------|
| `video_id`         | Уникальный идентификатор видео                | object    |
| `title`            | Название видео                                | object    |
| `duration_sec`     | Длительность видео в секундах (5–59)          | int64     |
| `hashtags_count`   | Количество хэштегов (0–9)                     | int64     |
| `views`            | Количество просмотров                         | int64     |
| `likes`            | Количество лайков                             | int64     |
| `comments`         | Количество комментариев                       | int64     |
| `shares`           | Количество репостов                           | int64     |
| `upload_hour`      | Час публикации (0–23)                         | int64     |
| `category`         | Категория контента                            | object    |

**Целевая переменная (создана в ходе работы):**  
`engagement_rate = (likes + comments + shares) / views`

## Выполненный анализ

### 1. Разведочный анализ данных (EDA)
- Загрузка данных и первичный осмотр структуры (`df.head()`, `df.info()`)
- Выделение числовых (`num_data`) и категориальных (`cat_data`) признаков
- Полные описательные статистики (`df.describe()`): count, mean, std, min, квартили, max
- Корреляционный анализ (Pearson) — большинство признаков имеют слабую линейную связь между собой

### 2. Визуализация данных
- Гистограммы + KDE для анализа распределений числовых признаков
- Boxplot и violinplot для выявления выбросов и сравнения по категориям
- Столбчатые диаграммы для сравнения метрик (просмотры, лайки, комментарии, shares, engagement_rate) между категориями
- Анализ зависимости просмотров и вовлечённости от часа публикации
- Анализ влияния количества хэштегов на ключевые метрики

### 3. Feature Engineering
- Создан бинарный признак `isDayTimeVideo` (видео, опубликованные с 12:00 до 18:00)
- Создана целевая переменная `engagement_rate`
- Подготовка данных для ML: удаление нерелевантных колонок, кодирование категориальных признаков (при необходимости)

### 4. Статистическое тестирование гипотез

Использованы тесты из `scipy.stats` при уровне значимости **α = 0.05**.

#### Гипотеза 1: Дневные видео набирают больше просмотров
- **Метод:** t-тест Уэлча (`ttest_ind` с `equal_var=False`)
- **Результат:** t-статистика ≈ 1.421, **p-value = 0.1568**
- **Вывод:** p-value > 0.05 → **нулевая гипотеза не отвергается**. Статистически значимых различий в просмотрах между дневными и ночными видео **не обнаружено**. Гипотеза **не подтверждена**.

#### Гипотеза 2: Видео категории Lifestyle чаще делятся
- **Метод:** Mann-Whitney U-тест
- **Результат:** U ≈ 5259, **p-value = 0.7611**
- **Вывод:** p-value >> 0.05 → **гипотеза не подтверждена**. Значимых различий в количестве репостов у Lifestyle по сравнению с другими категориями нет.

#### Гипотеза 3: Количество хэштегов влияет на просмотры
- **Метод:** Mann-Whitney U-тест (сравнение групп: < 5 и ≥ 5 хэштегов)
- **Результат:** U ≈ 11345, **p-value = 0.8110**
- **Вывод:** p-value > 0.05 → **гипотеза не подтверждена**. Количество хэштегов само по себе не оказывает статистически значимого влияния на просмотры.

### 5. Модели машинного обучения
Данные разделены на обучающую и тестовую выборки (test_size=0.2, random_state=42).

Сравнение трёх моделей регрессии:

| Модель                    | RMSE     | R²       | Качество                  | Интерпретация |
|---------------------------|----------|----------|---------------------------|---------------|
| **Linear Regression**     | 2.7664   | 0.0674   | Плохое                    | Объясняет лишь ~6.7% дисперсии. Слабая линейная зависимость в данных. |
| **Random Forest**         | 1.3740   | 0.7699   | Хорошее                   | Объясняет ~77% дисперсии. Значительно лучше линейной модели. |
| **Gradient Boosting**     | **1.2879** | **0.7979** | **Лучшее**              | Объясняет ~79.8% дисперсии. Наименьшая ошибка предсказания. |

**Вывод по моделям:** Нелинейные ансамблевые методы (Random Forest и особенно Gradient Boosting) существенно превосходят линейную регрессию. **Gradient Boosting** — лучшая модель для данной задачи.

### 6. Анализ важности признаков
На основе обученной модели **Gradient Boosting** построен график важности признаков (`feature_importances_`).

**Топ наиболее значимых признаков:**
- `views` (доминирует — ожидаемо, так как engagement_rate зависит от просмотров в знаменателе)
- `likes`
- `category`
- `comments` и `shares`

Менее важные: `duration_sec`, `hashtags_count`, `upload_hour`.

## Ключевые выводы и бизнес-инсайты

- **Comedy** и **Lifestyle** — самые вовлечённые категории (высокий `engagement_rate`).
- **Tech** показывает самые низкие показатели вовлечённости.
- Оптимальное время публикации — **13–14 часов** и **после 22:00**.
- Рекомендуемое количество хэштегов — **3–7** (качество важнее количества).
- Линейные модели плохо работают с этими данными. Для предсказания вовлечённости лучше использовать ансамблевые методы (Gradient Boosting / Random Forest).
- `views` — самый сильный предиктор `engagement_rate` (высокая корреляция с числителем формулы).
